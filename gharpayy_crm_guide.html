1. What is a CRM?

Simple Analogy

Imagine you run a property business. Every day, 50 people call asking about houses. You write each person's info on sticky notes. After a week — 350 sticky notes — total chaos!

A CRM (Customer Relationship Management) system is a digital notebook that stores all customer info, tracks where each customer is in the buying process, reminds you to follow up, and shows who's performing best.

4. React Fundamentals

4.1 Components

A component = reusable piece of UI. Like a cookie cutter — one shape, many cookies.

const StatCard = ({ title, value }) => {
  return (
    <div>
      <h3>{title}</h3>
      <p>{value}</p>
    </div>
  );
};

4.2 Props (Properties)

Analogy: Arguments you pass to a function. Like telling someone "make a card with name=Rahul, age=25".

// Parent PASSES props
<StatCard title="Total Leads" value={42} variant="primary" />

// Child RECEIVES props
const StatCard = ({ title, value, variant = "default" }) => { ... }
  • Props flow ONE WAY: parent → child
  • Props are read-only
  • variant = "default" = default value if not provided

4.3 State (useState)

Analogy: Your personal notepad — things you track and can change yourself.

const [search, setSearch] = useState("");
//      ↑            ↑                ↑
//  current value   updater fn      initial value

Analogy: Setting an alarm: "When I wake up (component loads), do these things."

useEffect(() => {
  fetchLeads();

Analogy: Write answer on paper instead of recalculating every time.

5. Component Deep Dive

5.1 StatCard.jsx

Purpose: Small card showing ONE metric ("Total Leads: 42").

VARIANTS Pattern (Config-driven styling)

const VARIANTS = {
  default: { card: "bg-white border-slate-200", icon: "bg-slate-100" },
  primary: { card: "bg-white border-blue-100",  icon: "bg-blue-50" },
  accent:  { card: "bg-white border-teal-100",  icon: "bg-teal-50" },
  warning: { card: "bg-white border-amber-100", icon: "bg-amber-50" },
};
const styles = VARIANTS[variant] ?? VARIANTS.default;

?? (Nullish Coalescing): If left side is null/undefined, use right side. Unlike ||, it doesn't treat 0 or "" as falsy.

5.2 AddLead.jsx

Purpose: Form to add leads with Smart Input that auto-parses pasted text.

Controlled Components

// React state controls the input value
<input
  value={form.name}
  onChange={e => setForm(f => ({ ...f, name: e.target.value }))}
/>

Spread operator ...f copies all fields, then overrides just one. Without it, you'd lose other fields!

Form Submission Flow

User clicks "Create Lead" ↓ handleSubmit(e) ↓ e.preventDefault() ← Stops page reload ↓ Validate name & phone ↓ submitting = true ← Disables button ↓ Append budget & location to notes ↓ addLead() → API call to backend ↓ Success: toast + navigate to /leads ↓ submitting = false

5.3 PipelineBoard.jsx

Purpose: Kanban board (like Trello). Drag leads between stage columns.

Uses Map to group leads by stage, useMemo for performance. See Section 7 for drag & drop details.

5.4 Leads.jsx

Purpose: Table view of ALL leads with search + filter.

const filtered = leads.filter((l) => {
  if (search && !name.includes(search)) return false;

6. State Management — Context API

The Problem: Prop Drilling

// WITHOUT Context — pass data through every level 😩
App → Dashboard → StatsRow → StatCard (3 levels!)

// WITH Context — any child accesses directly 🎉
CrmProvider → ANY component calls useCrm()

How CrmContext.jsx Works

// Step 1: Create context (empty bulletin board)
const CrmContext = createContext();

// Step 2: Provider fills it with data
export const CrmProvider = ({ children }) => {
  const [leads, setLeads] = useState([]);
  return (
    <CrmContext.Provider value={{ leads, fetchLeads, addLead }}>
      {children}
    </CrmContext.Provider>
  );
};

// Step 3: Custom hook (shortcut)
export const useCrm = () => useContext(CrmContext);

// Step 4: Use anywhere!
const { leads, addLead } = useCrm();
Use Context for simple state. Use Redux for complex apps with many state updates, middleware needs, and large teams.

7. Drag & Drop — PipelineBoard

Analogy: Card game — pick up a card (dragStart), move over a pile (dragOver), put it down (drop).

EventWhenWhat Happens
onDragStartPick up carde.dataTransfer.setData('leadId', id)
onDragOverHover over columne.preventDefault() — allows dropping
onDropRelease cardRead ID, update status via API
const handleDrop = (e, status) => {
  e.preventDefault();
  const leadId = e.dataTransfer.getData('leadId');
  if (leadId) updateStatus(leadId, status);  // API + state update
  setDragOverStage(null);
};
e.preventDefault() in onDragOver is MANDATORY. Browsers don't allow dropping by default. This is the #1 beginner mistake!

8. Regex & Smart Input Parsing

Regex = Search pattern. Like telling someone "find all words starting with R" — but way more powerful.

Phone Number Regex

/\b(?:\+91[\s-]?)?([5-9]\d{9})\b/
PartMeaning
\bWord boundary — prevents matching inside larger numbers
(?:\+91[\s-]?)?Optional +91 prefix with optional space/dash
[5-9]First digit: 5,6,7,8, or 9 (Indian mobile)
\d{9}Exactly 9 more digits

Position-Based Parsing Strategy

"Rahul Sharma 9876543210 2BHK Koramangala 1200000" ↑ BEFORE phone ↑ ↑ phone ↑ ↑ AFTER phone ↑ Step 1: Find phone → "9876543210" Step 2: Before phone → "Rahul Sharma" → NAME Step 3: After phone → find budget "1200000" Step 4: Remaining → "2BHK Koramangala" → LOCATION

Q2: What are controlled components?

A controlled component's value is managed by React state, not the DOM. The input shows what state tells it. Benefits: single source of truth, easy validation, easy reset.

Q3: Explain useEffect with examples from this codebase.

In Leads.jsx: useEffect(() => { fetchLeads(); fetchAgents(); }, [])
Empty [] = run ONCE on mount. Used for initial data fetching. Without [] = infinite loop!

Q4: Context API vs Redux?

Context: Simple state, small apps, no middleware needed. Redux: Complex state, large teams, need devtools/middleware. GharPayy uses Context because state is simple.

Q5: Explain the CRM architecture.

"3-tier: Presentation (React), Business Logic (Express), Data (MongoDB). Frontend talks to backend via REST APIs. Uses Context for state management to avoid prop drilling."

Q7: Drag-and-drop doesn't work. How to debug?
  1. Check onDragOver has e.preventDefault()
  2. Check dataTransfer.getData returns correct ID
  3. Check Network tab — is API call made? Response?
  4. Check updateStatus — does state update correctly?
  5. Check MongoDB — did status change in DB?
Q8: How would you add "filter by date range"?
  1. Add startDate & endDate state
  2. Add date input fields in filter bar
  3. In filter logic: check createdAt is within range
  4. Consider debouncing for large datasets

10. Common Beginner Mistakes

❌ Mistake 1: Forgetting e.preventDefault()

Wrong: Form submits → page reloads → data lost!

Fix: Add e.preventDefault() at the start of handleSubmit

❌ Mistake 2: Mutating State Directly
// ❌ WRONG — React won't re-render!
form.name = "Rahul";

// ✅ CORRECT — Create new object
setForm(f => ({ ...f, name: "Rahul" }));

React compares by reference. Same object = "nothing changed".

❌ Mistake 3: Missing key prop in lists
// ❌ {leads.map(l => <Card lead={l} />)}
// ✅ {leads.map(l => <Card key={l._id} lead={l} />)}

Without key, React re-renders the ENTIRE list.

❌ Mistake 4: useEffect without dependencies
// ❌ INFINITE LOOP! Runs → fetches → state changes → runs again...
useEffect(() => { fetchLeads(); });

// ✅ Runs only once
useEffect(() => { fetchLeads(); }, []);
❌ Mistake 5: No loading/error handling
// ✅ Production-ready pattern:
setSubmitting(true);
try {
  await api.createLead(form);
  toast.success('Created!');
} catch (err) {
  toast.error('Failed!');
} finally {
  setSubmitting(false);
}
❌ Mistake 6: Using || instead of ??
// ❌ 0 || "N/A" = "N/A" (0 is falsy!)
// ✅ 0 ?? "N/A" = 0    (?? only checks null/undefined)
For interviews, always explain WHY you made a choice: "I used Context API instead of Redux because the state is simple — adding Redux would be over-engineering."
